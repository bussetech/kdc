The KDC feedback line is a phone number you can call to leave a short
recorded message about a data-center correction or a source we should be
tracking. It is a voicemail line, not a live conversation. No one picks
up, and there is no bot to talk to on the call.

After you leave a message, a gnome reads the transcript and drafts a
correction or a source entry. Nothing you say is published straight to
the site. A person reviews the draft before any record changes.

## Not live yet

The line is not switched on. When it goes live, the number will be
printed here. For now the number is coming soon.

## Access code

A public phone number costs us on every call and attracts junk, so the
line is gated by an access code. If you have been given a code, enter it
on your keypad when you call, then press the pound key. You get three
tries. If none of them match, the line asks you to check the code and
call back later.

## If the mailbox is full

The line holds a limited number of messages each day. This cap keeps the
running cost predictable. If the day's mailbox is already full when you
call, the line kindly asks you to try back tomorrow, and the call ends
without recording.

## Recordings and your privacy

Recordings are kept for 30 days and then deleted. We treat your phone
number as private and mask it in our records. The message you leave
becomes a draft that a human reviews before anything in the dataset
changes.

## Contribute in writing

You can send the same thing in writing. Open a data-correction or a
source-suggestion issue on this project's GitHub repository, and it lands
in the same review queue as a phoned-in message.
